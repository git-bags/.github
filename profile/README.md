## How it works

1. **Install** — GitBags installs as a GitHub App on your repo
2. **Track** — It tracks all contributions: PRs, commits, code reviews, and issues
3. **Launch** — A token is launched on [<img src="https://raw.githubusercontent.com/git-bags/.github/main/bag.png" width="14" height="14"> bags.fm](https://bags.fm/6D9R28NvyGMiSknovuetmgzzpxqoSs72xie6GRuABAGS) for each repo
4. **Earn** — 1% of every trade becomes contributor fees
5. **Claim** — Fees are distributed proportionally based on contribution weight
6. **Cash out** — Contributors sign in with GitHub, connect a Solana wallet, and claim fees instantly on-chain

## Configuration

Customize fee distribution with a `.gitbags.yaml` in your repo root:

```yaml
# .gitbags.yaml
weights:
  commits: 40
  pull_requests: 30
  reviews: 20
  issues: 10
```

## Why GitBags?

Open-source contributors rarely get rewarded for their work. Maintainers burn out. Projects go unfunded.

GitBags fixes this — every trade on your repo's token sends fees directly to the people writing the code. No grants, no donations, no middlemen. Just code → rewards.
